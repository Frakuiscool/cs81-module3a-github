What did you learn about using Git and GitHub?
I learned how to make a GitHub repository, clone it to my computer, and use basic Git commands like add, commit, and push to save and upload my code. I also learned how to write a README to explain what my code does.
What was most confusing or frustrating?
The Git commands themselves weren't too bad, but I ran into some annoying terminal problems. I had trouble finding the copy button on GitHub, then pasting the link into my terminal since normal Ctrl+V didn't work. I also got stuck when cd Desktop didn't work because of a folder naming issue.
Why do you think developers use Git?
Developers use Git so they can track what changes were made and when, using commit messages. If something breaks, they can look back and see what changed. It also lets multiple people work on the same project without overwriting each other's work.
How might this help with larger projects in the future?
Making small, clear commits would make it easier to find bugs later, since I could see exactly where something changed. It would also help me work with other people on bigger projects without messing up each other's code. This assignment gave me a basic feel for a workflow I'll probably use a lot going forward.
